# Реализация хеш-таблицы, множества и мультимножества на языке C

## Краткое описание решения

В проекте реализованы три основные структуры данных:

1. **Хеш-таблица**
   Базовая структура данных, которая хранит пары вида:

   ```text
   ключ -> значение
   ```

   В качестве ключей используются строки, а значения хранятся в виде указателей `void*`. Такой подход делает хеш-таблицу универсальной: она может хранить значения разных типов, если пользователь сам правильно приводит указатели к нужному типу.

2. **Множество**
   Множество реализовано поверх хеш-таблицы. В множестве важен только факт наличия элемента, поэтому элемент хранится как ключ, а значение является фиктивным.

   Пример:

   ```text
   cat
   dog
   bird
   ```

   Если один и тот же элемент добавить несколько раз, он всё равно будет храниться только один раз.

3. **Мультимножество**
   Мультимножество также реализовано поверх хеш-таблицы, но в качестве значения хранится счётчик повторений элемента.

   Пример:

   ```text
   cat -> 3
   dog -> 2
   bird -> 1
   ```

   Таким образом, мультимножество позволяет не только проверить наличие элемента, но и узнать, сколько раз он был добавлен.

## Реализация хеш-таблицы

Хеш-таблица состоит из массива элементов `entry`. Каждый элемент содержит:

```c
const char* key;
void* value;
```

Основные операции хеш-таблицы:

* создание таблицы;
* удаление таблицы;
* добавление элемента;
* получение значения по ключу;
* удаление элемента;
* получение количества элементов;
* обход элементов с помощью итератора.

При добавлении нового элемента строковый ключ копируется с помощью `strdup`, поэтому таблица хранит собственную копию ключа.

## Обработка коллизий

Для обработки коллизий используется метод **открытой адресации с линейным пробированием**.

Коллизия возникает, когда разные ключи дают одинаковый индекс в массиве. В этом случае программа проверяет следующую ячейку массива. Если она тоже занята, поиск продолжается дальше, пока не будет найдено свободное место или нужный ключ.

Схема работы:

```text
индекс занят -> перейти к следующему индексу
следующий занят -> перейти дальше
найдено свободное место -> вставить элемент
```

При достижении конца массива поиск продолжается с начала массива.

Для удаления используется специальная метка удалённого элемента. Это нужно для корректной работы линейного пробирования: если просто поставить `NULL`, поиск элементов, которые были вставлены после удалённого ключа, может сломаться.

## Расширение таблицы

При заполнении таблицы более чем наполовину выполняется расширение массива. Новая вместимость становится в два раза больше предыдущей.

После расширения все существующие элементы переносятся в новый массив с повторным вычислением индексов.

Это нужно, чтобы уменьшить количество коллизий и сохранить эффективную работу операций вставки, поиска и удаления.

## Выбор хеш-функции

В проекте используется хеш-функция **FNV-1a** в 64-битном варианте. Это простая и быстрая хеш-функция, хорошо подходящая для учебной реализации хеш-таблицы со строковыми ключами.

Алгоритм FNV-1a работает следующим образом: сначала значение хеша инициализируется специальной константой `FNV_OFFSET`. Затем строка обрабатывается посимвольно. Для каждого байта выполняется операция XOR между текущим значением хеша и значением байта, после чего результат умножается на специальное простое число `FNV_PRIME`.

Выбор FNV-1a обусловлен несколькими причинами. Во-первых, функция проста в реализации и не требует дополнительных библиотек. Во-вторых, она быстро работает, так как использует только проход по строке, операцию XOR и умножение. В-третьих, FNV-1a обеспечивает достаточно хорошее распределение хешей для коротких и средних строковых ключей, что подходит для демонстрационной реализации хеш-таблицы.

В данной работе используется 64-битный вариант FNV-1a. Он выбран потому, что современные системы обычно хорошо работают с 64-битными целыми числами, а также потому, что 64-битный хеш уменьшает вероятность случайных совпадений значений хеша по сравнению с 32-битным вариантом. Это особенно полезно, если хеш-таблица будет содержать большое количество элементов.

При этом важно учитывать, что FNV-1a не является криптографической или рандомизированной хеш-функцией. Это означает, что она не предназначена для защиты данных и не должна использоваться в задачах безопасности. Теоретически злоумышленник может подобрать набор ключей, которые будут создавать большое количество коллизий, из-за чего операции поиска, вставки и удаления могут замедлиться. Однако в рамках данного учебного проекта такая угроза не является критичной, так как хеш-таблица используется для демонстрации принципов работы структуры данных, а не для обработки недоверенных внешних данных.

Таким образом, FNV-1a является подходящим выбором для данной реализации: она простая, быстрая, понятная и достаточно эффективная для учебной хеш-таблицы со строковыми ключами.


## Получение индекса

После вычисления хеша индекс в массиве получается так:

```c
index = hash & (capacity - 1);
```

Такой способ работает корректно, потому что вместимость таблицы является степенью двойки. Использование битовой операции `&` быстрее, чем операция взятия остатка `%`.

Например, если вместимость таблицы равна `16`, то индекс будет находиться в диапазоне от `0` до `15`.

## Реализация множества

Множество реализовано как оболочка над хеш-таблицей.

Структура множества:

```c
typedef struct hash_set {
    hash_table* table;
} hash_set;
```

В множестве элемент хранится как ключ. Значение не имеет смысла, поэтому используется фиктивное значение.

Основные операции множества:

* создание множества;
* добавление элемента;
* проверка наличия элемента;
* удаление элемента;
* получение размера множества;
* обход элементов.

Пример логики:

```text
hash_set_add       -> добавляет ключ в хеш-таблицу
hash_set_contains  -> ищет ключ в хеш-таблице
hash_set_remove    -> удаляет ключ из хеш-таблицы
hash_set_size      -> возвращает количество ключей
```

Множество удобно использовать, когда нужно хранить только уникальные элементы.

## Реализация мультимножества

Мультимножество реализовано как оболочка над хеш-таблицей.

Структура мультимножества:

```c
typedef struct hash_multiset {
    hash_table* table;
    size_t total_count;
} hash_multiset;
```

В мультимножестве ключом является элемент, а значением — указатель на счётчик типа `size_t`.

Пример:

```text
cat -> 3
dog -> 2
bird -> 1
```

Основные операции мультимножества:

* создание мультимножества;
* добавление элемента;
* удаление одного вхождения элемента;
* удаление всех вхождений элемента;
* получение количества повторений элемента;
* получение количества уникальных элементов;
* получение общего количества элементов;
* обход элементов.

Мультимножество удобно использовать для подсчёта повторений, например, для подсчёта количества одинаковых слов в тексте.
